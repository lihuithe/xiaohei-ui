# 226 消息/通知流布局（Message & Notification Stream Layout）

## 功能名称

消息/通知流布局 — 提供通知中心、活动流、评论线程三种可复用骨架。

## 功能背景/动机

现有脚手架已有 `ChatLayout.vue`（一对一聊天界面）和 `002-notification-center`（应用内通知中心的功能设计），但缺少**通用的消息/通知流页面级布局模板**。在桌面应用中，消息流不仅限于聊天，还包括：系统通知列表、用户活动时间线、文档评论线程、操作日志流等。这些场景共享相似的结构：按时间倒序排列的条目、未读标记、头像/图标、操作按钮、以及加载更多/实时更新的交互。

本功能提供三种消息流的布局模板，强调"流式"内容的编排规范，与具体的通信业务解耦。

## 功能描述

包含以下三种消息流布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `NotificationCenterLayout` | 通知中心：分组（今日/更早）+ 未读标记 + 一键已读 + 操作菜单 | 系统通知、应用内消息、提醒列表 |
| `ActivityStreamLayout` | 活动流：时间戳 + 用户头像 + 操作描述 + 对象引用 | 操作审计、团队协作动态、 feed 流 |
| `CommentThreadLayout` | 评论线程：嵌套回复 + 输入框 + 点赞/回复操作 | 文档评论、任务讨论、评审反馈 |

每种布局均内建：
- 按时间分组（今天、昨天、本周、更早）
- 未读状态指示（圆点、背景高亮）
- 空状态（`Empty` 组件："暂无消息"）
- 加载更多/实时更新的占位交互
- 与 `Avatar`、`Badge`、`Empty`、`ScrollArea` 的衔接

## 目标用户

- 需要实现通知中心、活动动态、评论功能的开发者
- 希望统一消息流式内容展示风格的开发者

### 涉及的技术点

- 使用现有 `Avatar`、`Badge`、`Button`、`Empty`、`ScrollArea` 组件
- 使用 `StaggerList`（已有）或 `AnimatedTransition` 实现新消息的进入动画
- 使用 `Item` 组件族（`Item`、`ItemHeader`、`ItemContent`、`ItemFooter`）构建流式条目
- 使用 `Textarea` + `Button` 构建评论输入区
- 使用 `virtual-scroller` 技术（如数据量大时）通过 `#virtual` slot 提供虚拟滚动接入点

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/NotificationCenterLayout.vue`
- 新增 `src/components/layouts/ActivityStreamLayout.vue`
- 新增 `src/components/layouts/CommentThreadLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- `NotificationCenterLayout` 可与 `SideDrawerLayout`（210）组合（通知作为右侧抽屉）
- `ActivityStreamLayout` 可与 `TimelineVerticalLeft`（203）组合（活动流使用时间线样式）
- `CommentThreadLayout` 可与 `DetailStackedLayout`（201）组合（评论作为详情页底部区块）

**Props 设计（以 NotificationCenterLayout 为例）：**

```ts
interface NotificationItem {
  id: string
  title: string
  description?: string
  timestamp: Date | string
  read: boolean
  icon?: Component
  avatar?: string
  actions?: {
    label: string
    onClick: () => void
  }[]
}

interface NotificationCenterLayoutProps {
  items: NotificationItem[]
  groupByDate?: boolean      // 是否按日期分组，默认 true
  showMarkAllRead?: boolean  // 是否显示"全部已读"，默认 true
  loading?: boolean
  hasMore?: boolean          // 是否还有更多，用于加载更多
}

// Events
// @mark-read     — 标记单条已读
// @mark-all-read — 标记全部已读
// @load-more     — 加载更多
// @action-click  — 点击某条通知的操作按钮
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `NotificationCenterLayout` 的未读项左侧显示圆点指示，背景色与已读项有微妙差异
- [ ] 按日期分组时，分组标题（"今日"、"昨天"）在滚动时保持 sticky 定位
- [ ] `CommentThreadLayout` 的嵌套回复最多展示 3 级，更深层级缩进不再增加
- [ ] 新消息进入时有滑入动画（从右侧或底部），与 `AnimatedTransition` 协调
- [ ] 评论输入框自动增高（`Textarea` 的 auto-resize），最大高度不超过 200px
- [ ] 空状态时展示 `Empty` 占位（铃铛图标 + "暂无通知"文案）
- [ ] 所有时间戳支持相对时间显示（"2 分钟前"），超过 7 天显示绝对日期

## 优先级

P1 — 消息流是协作类、通知类桌面应用的核心布局。
