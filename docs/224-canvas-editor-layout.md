# 224 画布/图片编辑器布局占位（Canvas Editor Layout）

## 功能名称

画布/图片编辑器布局占位 — 提供经典画布编辑器、无边框沉浸画布、双栏素材+画布三种可复用骨架。

## 功能背景/动机

桌面应用中大量场景需要"画布"作为核心交互区域：图片标注、流程图绘制、白板协作、设计稿预览、PDF 批注等。现有脚手架已有 `AspectRatio` 组件，但缺少**面向画布编辑场景的页面级布局模板**。画布布局有其特殊性：需要最大化利用屏幕空间、工具栏需要围绕画布分布、属性面板需要精确调整元素属性。

本功能提供纯粹的 UI 布局骨架，不绑定任何具体的画布渲染引擎（如 Canvas 2D、SVG、WebGL），只提供编辑器的"外壳"布局，让开发者可以即插即用地接入任意渲染技术。

## 功能描述

包含以下三种画布编辑器布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `ClassicCanvasLayout` | 顶部主工具栏 + 左侧工具箱 + 中央画布 + 右侧属性面板 | 图片编辑器、流程图工具、设计工具 |
| `ImmersiveCanvasLayout` | 无边框沉浸模式：悬浮工具栏 + 全屏画布 + 可唤起的侧边面板 | 白板、演示模式、聚焦创作 |
| `AssetCanvasLayout` | 左侧素材库 + 中央画布 + 底部时间轴/图层区 | 幻灯片制作、动画编辑器、视频剪辑 |

每种布局均内建：
- 画布容器的自适应尺寸（占满可用空间，保持比例或拉伸）
- 工具栏/面板的展开/收起控制
- 缩放控制占位（放大/缩小/适应屏幕/原始比例）
- 画布坐标/尺寸状态显示（底部状态栏）
- 暗色/亮色主题适配（画布区通常使用中性灰背景）

## 目标用户

- 需要集成画布、白板、图片编辑、流程图功能的开发者
- 需要最大化画布利用率的创作类桌面应用开发者

### 涉及的技术点

- 使用 CSS `flex` + `overflow-hidden` 实现画布区域的自适应占满
- 使用 `ResizablePanelGroup` 实现左/右/底层面板的拖拽调宽
- 使用 `ui/tooltip/` 为工具栏按钮提供提示
- 使用 `ui/tabs/` 在素材库和属性面板内切换分类
- 使用 `ScrollArea` 处理素材列表和图层列表的独立滚动
- 画布容器使用 `relative` 定位，内部渲染层使用 `absolute` 定位

**Props 设计（以 ClassicCanvasLayout 为例）：**

```ts
interface ClassicCanvasLayoutProps {
  canvasWidth?: number       // 画布逻辑宽度（仅用于状态显示）
  canvasHeight?: number      // 画布逻辑高度
  zoom?: number              // 当前缩放比例，默认 100
  minZoom?: number           // 最小缩放，默认 10
  maxZoom?: number           // 最大缩放，默认 500
  showRulers?: boolean       // 是否显示标尺，默认 false
  showGrid?: boolean         // 是否显示网格，默认 false
}

// Slot 约定
// #toolbar       — 顶部主工具栏
// #toolbox       — 左侧工具箱
// #canvas        — 画布渲染区域（必填）
# #properties    — 右侧属性面板
// #status-bar    — 底部状态栏（覆盖默认的缩放/尺寸信息）
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] 画布区域在窗口大小变化时自动占满可用空间，无固定尺寸限制
- [ ] `ClassicCanvasLayout` 的左/右/底层面板支持拖拽调整宽度/高度
- [ ] `ImmersiveCanvasLayout` 的悬浮工具栏在鼠标离开 2 秒后自动隐藏，hover 边缘时 200ms 内唤起
- [ ] 画布容器的背景色使用中性灰（`bg-neutral-100` 或 `bg-neutral-900`），与工具栏形成对比
- [ ] 缩放控制支持通过 props 传入当前比例，通过事件通知比例变化
- [ ] 标尺和网格线（如启用）使用 1px 虚线，颜色对比度适中
- [ ] 所有布局不绑定具体渲染引擎，`#canvas` slot 可放置任意 Vue 组件或原生 Canvas

## 优先级

P1 — 画布布局是创作类、设计类桌面应用的核心，通用性强。
