# 231 可拖拽排序列表布局（Draggable Sort List Layout）

## 功能名称

可拖拽排序列表布局 — 提供垂直排序列表、网格排序、双栏穿梭框三种可复用骨架。

## 功能背景/动机

桌面应用中，用户经常需要调整项目的顺序或分组：任务优先级排序、步骤流程编排、分类顺序调整、字段顺序自定义等。现有脚手架已有 `004-file-drag-drop`（文件拖拽处理的功能设计），但它是针对文件拖拽的特定场景，缺少**通用的、可复用的拖拽排序列表布局模板**。开发者在实现排序功能时，需要自行处理拖拽手柄、拖拽时的视觉反馈、放置区域的占位符、以及排序完成后的状态更新。

本功能提供三种拖拽排序的布局模板，只提供 UI 骨架和拖拽占位槽，不绑定具体的拖拽库（如 `sortablejs`、`@vueuse/motion`），让开发者按需接入。

## 功能描述

包含以下三种拖拽排序布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `VerticalSortListLayout` | 垂直列表拖拽排序：每行有拖拽手柄，拖拽时有占位符提示 | 任务列表、菜单排序、步骤编排 |
| `GridSortLayout` | 网格拖拽排序：卡片在网格内自由拖拽换位 | 仪表盘 widget 排序、图片排序、卡片优先级 |
| `ShuttleBoxLayout` | 双栏穿梭框：左右两栏，项目可拖拽或按钮转移至另一栏 | 权限分配、分类归属、字段选择 |

每种布局均内建：
- 拖拽手柄的视觉标识（六点/四箭头图标）
- 拖拽中的视觉反馈（半透明、阴影放大、原位置占位符）
- 放置区域的视觉提示（边框高亮、背景色变化）
- 空栏状态（`Empty` 组件："拖拽项目到此处"）
- 与 `Item`、`Card`、`Empty` 的衔接

## 目标用户

- 需要实现列表排序、卡片重排、双栏穿梭功能的开发者
- 希望提供直观的拖拽交互体验的开发者

### 涉及的技术点

- 使用现有 `Item`、`Card`、`Empty`、`Button` 组件构建列表项
- 使用 `lucide-vue-next` 的 `GripVertical` 或 `Move` 图标作为拖拽手柄
- 拖拽手柄区域使用 `cursor-grab` / `cursor-grabbing` 样式反馈
- 拖拽中的元素使用 `opacity-50` + `shadow-lg` + `scale-[1.02]` 视觉反馈
- 占位符使用 `border-2 border-dashed border-primary` 虚线边框
- 使用 HTML5 Drag and Drop API 或提供 `draggable` slot 属性接入第三方库

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `VerticalSortListLayout` 的拖拽手柄在 hover 时显示为 `grab`，拖拽时变为 `grabbing`
- [ ] 拖拽中的元素有半透明 + 阴影放大的视觉反馈
- [ ] 原位置显示虚线边框占位符，高度与被拖拽元素一致
- [ ] `ShuttleBoxLayout` 支持拖拽转移和按钮转移两种方式
- [ ] 空栏时显示 `Empty` 占位（"拖拽项目到此处"）
- [ ] 排序完成后触发 `@sort` 事件，返回新的顺序数组
- [ ] 所有布局不绑定具体拖拽库，通过 `draggable` 属性和事件委托实现基础交互

## 优先级

P1 — 拖拽排序是配置类、管理类桌面应用的高频交互。
