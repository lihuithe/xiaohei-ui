# 222 批量操作表单布局（Batch Action Form Layout）

## 功能名称

批量操作表单布局 — 提供批量编辑、批量确认、批量导入向导三种可复用骨架。

## 功能背景/动机

桌面应用中，用户对列表数据的批量处理需求非常普遍：批量修改状态、批量删除、批量导入数据等。现有脚手架已有 `Checkbox` 组件和 `DataTableLayout`（含行选择功能），但缺少**面向批量操作的页面级布局模板**。开发者在实现批量操作时，需要自行处理"选择态 → 操作态"的切换、批量表单字段的展示、确认对话框的结构、以及导入向导的步骤编排。

本功能提供三种批量操作的布局模板，覆盖"修改"、"确认"、"导入"三大场景，确保批量操作的交互体验一致性。

## 功能描述

包含以下三种批量操作布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `BatchEditLayout` | 批量编辑：顶部已选摘要 + 表单字段（仅展示共性的字段） | 批量修改状态、标签、负责人 |
| `BatchConfirmLayout` | 批量确认：操作影响摘要 + 确认清单 + 二次确认按钮 | 批量删除、批量归档、批量导出 |
| `BatchImportWizardLayout` | 批量导入：文件上传 → 数据映射 → 预览 → 确认导入 | 数据迁移、批量创建、Excel 导入 |

每种布局均内建：
- 已选项目摘要（数量、名称预览、展开查看全部）
- 操作影响提示（"这将影响 24 个项目"）
- 表单验证（批量字段的必填/格式校验）
- 进度反馈（导入向导的进度条）
- 与 `Table`、`Card`、`Empty` 的衔接

## 目标用户

- 需要实现列表批量编辑、批量删除、数据导入的开发者
- 希望降低批量操作误触风险、提升操作确定性的开发者

### 涉及的技术点

- 使用现有 `Stepper` 组件实现导入向导的步骤指示器
- 使用 `Table`、`Checkbox`、`Select`、`Input` 构建表单和清单
- 使用 `Alert` 组件展示操作警告（不可撤销提示）
- 使用 `Progress` 组件展示导入进度
- 使用 `Empty` 组件处理无匹配字段或空数据预览
- 使用 `AnimatedTransition` 实现步骤切换的淡入动画

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/BatchEditLayout.vue`
- 新增 `src/components/layouts/BatchConfirmLayout.vue`
- 新增 `src/components/layouts/BatchImportWizardLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出

**Props 设计（以 BatchConfirmLayout 为例）：**

```ts
interface BatchItem {
  id: string
  title: string
  subtitle?: string
}

interface BatchConfirmLayoutProps {
  items: BatchItem[]
  actionTitle: string
  actionDescription: string
  confirmText?: string      // 需要用户输入的确认文本，如 "DELETE"
  destructive?: boolean     // 是否为破坏性操作，默认 false
  showItemList?: boolean    // 是否展示完整项目清单，默认 true
}

// Events
// @confirm — 用户确认操作
// @cancel  — 用户取消操作
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `BatchEditLayout` 的字段默认值均为"保持原值"，不修改留空字段
- [ ] `BatchConfirmLayout` 的破坏性操作（如删除）默认需要二次确认（输入确认文本或勾选确认框）
- [ ] `BatchImportWizardLayout` 的步骤切换时，未完成的必填步骤阻止进入下一步
- [ ] 已选项目摘要支持展开/收起，展开后展示完整项目列表（带滚动）
- [ ] 批量操作进行中时，确认按钮显示 loading 状态并禁用
- [ ] 操作完成后展示 `ResultSuccessLayout`（206）或 `ResultErrorLayout`（206）作为反馈
- [ ] 所有文案支持通过 props 传入，便于国际化

## 优先级

P1 — 批量操作是数据管理类桌面应用的高频场景，能显著提升操作效率。
