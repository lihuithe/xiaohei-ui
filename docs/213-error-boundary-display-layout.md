# 213 错误边界展示模板（Error Boundary Display Layout）

## 功能名称

错误边界展示模板 — 提供应用级错误页、区块级错误卡片、优雅降级三种错误边界展示骨架。

## 功能背景/动机

现有脚手架已有 `ErrorBoundary.vue`（基础的错误捕获 + 错误信息展示），但视觉表现较为简陋：红色背景框 + 原始错误堆栈。对于面向用户的桌面应用，这种展示过于技术化，容易引起用户恐慌。同时，`ErrorBoundary.vue` 仅提供单一的全局样式，缺少**针对不同上下文（整个应用崩溃 vs. 某个区块失败）的差异化布局模板**。

本功能在不修改现有 `ErrorBoundary.vue` 的前提下，提供三种更优雅、更友好的错误展示布局，让开发者可以根据错误影响范围选择合适的展示方式。

## 功能描述

包含以下三种错误边界展示变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `AppCrashLayout` | 全屏错误页，应用完全无法继续运行 | 渲染根组件失败、关键依赖缺失 |
| `BlockErrorLayout` | 局部区块错误卡片，不影响应用其他部分 | 某个 widget、图表、列表加载失败 |
| `GracefulDegradeLayout` | 降级展示：移除失败功能，保留可用部分 | 非核心模块失败时的兜底策略 |

每种布局均内建：
- 友好的错误说明（非技术语言）
- 重试按钮（与 `ErrorBoundary` 的 `reset` 衔接）
- 可选的"报告问题"占位按钮
- 暗色/亮色主题适配
- 与 `ResultErrorLayout`（206）的视觉语言一致

## 目标用户

- 需要为应用不同层级提供差异化错误展示的开发者
- 希望降低用户面对错误时焦虑感的开发者

## 详细设计

### 布局结构描述

#### 1. 应用级崩溃页（AppCrashLayout）

```
+--------------------------------------------------+
|                                                  |
|                                                  |
|              +------------------+                |
|              |   ⚠️ (大图标)     |                |
|              |   柔和色调背景    |                |
|              +------------------+                |
|                                                  |
|              应用遇到了问题                      |
|         我们无法加载必要的数据，请稍后再试       |
|                                                  |
|         [  重新加载应用  ]  [  查看详情  ]        |
|                                                  |
|         如果问题持续存在，请联系支持团队         |
|                                                  |
|         ┌─────────────────────────────────┐     |
|         │ 错误详情（可折叠，默认收起）     │     |
|         │ TypeError: Cannot read...       │     |
|         └─────────────────────────────────┘     |
|                                                  |
+--------------------------------------------------+
```

#### 2. 局部区块错误卡片（BlockErrorLayout）

```
+--------------------------------------------------+
| 仪表盘                                           |
+--------------------------------------------------+
| ┌────────────┐ ┌────────────┐ ┌────────────┐   |
| │ 统计卡片   │ │ 统计卡片   │ │ 统计卡片   │   |
| └────────────┘ └────────────┘ └────────────┘   |
| ┌────────────────────────────────────────────┐  |
| │  ⚠️  近期活动加载失败                       │  |
| │      无法连接到数据源                       │  |
| │                              [重新加载]     │  |
| └────────────────────────────────────────────┘  |
|  ↑ 仅该区块显示错误，其余内容正常展示            |
+--------------------------------------------------+
```

#### 3. 优雅降级（GracefulDegradeLayout）

```
+--------------------------------------------------+
| 项目详情                                         |
+--------------------------------------------------+
| ┌────────────────────────────────────────────┐  |
| │ 项目名称: Demo App                          │  |
| │ 负责人: 张三                                │  |
| │ 标签: [设计中] [进行中]                     │  |
| └────────────────────────────────────────────┘  |
|                                                  |
| ┌────────────────────────────────────────────┐  |
| │  项目统计图表                               │  |
| │  ⚠️ 图表加载失败，已显示基础数据             │  |
| │  任务数: 24 | 完成: 18 | 延期: 2            │  |
| └────────────────────────────────────────────┘  |
|  ↑ 图表失败，但用文字摘要替代，页面整体可用      |
+--------------------------------------------------+
```

### 涉及的技术点

- 使用现有 `ErrorBoundary.vue` 的 `onErrorCaptured` 和 `reset` 机制
- 使用 `ResultErrorLayout`（206）的视觉规范（图标、按钮、颜色）
- 使用 `Card`、`Alert`、`Button`、`Collapsible` 组件构建错误卡片
- 使用 `AnimatedTransition` 为错误状态的出现添加淡入动画
- 可选：集成 `vue-i18n` 的错误文案国际化

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/AppCrashLayout.vue`
- 新增 `src/components/layouts/BlockErrorLayout.vue`
- 新增 `src/components/layouts/GracefulDegradeLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- `AppCrashLayout` 可作为 `ErrorBoundary.vue` 的默认 slot 内容替换
- `BlockErrorLayout` 可嵌入 `DashboardLayout`、`DetailStackedLayout` 的任意区块

### 需要新增/修改的文件

```
desktop-app/src/components/layouts/
  ├── AppCrashLayout.vue            # 应用级崩溃页
  ├── BlockErrorLayout.vue          # 局部区块错误卡片
  ├── GracefulDegradeLayout.vue     # 优雅降级布局
  └── index.ts                      # 追加导出
```

**Props 设计（以 BlockErrorLayout 为例）：**

```ts
interface BlockErrorLayoutProps {
  title?: string           // 错误标题，默认"加载失败"
  description?: string     // 错误描述
  error?: Error            // 原始错误对象（用于详情展示）
  showRetry?: boolean      // 是否显示重试按钮，默认 true
  showDetails?: boolean    // 是否显示错误详情折叠区，默认 false
}

// Events
// @retry — 用户点击重试
// @report — 用户点击报告问题
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `AppCrashLayout` 的内容区在窗口任意高度下保持垂直居中
- [ ] `BlockErrorLayout` 的错误卡片不破坏父容器的布局流（即周围内容正常排列）
- [ ] `GracefulDegradeLayout` 的降级内容区支持通过 `#fallback` slot 自定义
- [ ] 重试按钮点击后触发 `retry` 事件，父组件可重新加载数据
- [ ] 错误详情折叠区默认收起，展开后显示原始错误消息和堆栈
- [ ] 所有文案支持通过 props 传入，便于国际化替换
- [ ] 暗色模式下错误图标的背景色自动适配（不使用硬编码的红色背景）

## 优先级

P1 — 错误处理是桌面应用健壮性的体现，与现有 `ErrorBoundary` 和 `105` 形成互补。

## 参考实现

- Slack 的"连接中断"提示条（BlockError 风格）
- Twitter/X 的"出错了，请稍后再试"局部卡片
- macOS 应用的崩溃报告对话框（AppCrash 风格）
- Chrome 的"无法访问此网站"页面（简约 + 可操作）
