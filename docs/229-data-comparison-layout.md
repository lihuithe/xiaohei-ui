# 229 数据对比布局（Data Comparison Layout）

## 功能名称

数据对比布局 — 提供左右对比、差异高亮对比、表格行对比三种可复用骨架。

## 功能背景/动机

桌面应用中，数据对比是常见需求：版本对比、文件差异、配置对比、前后状态对照等。现有脚手架已有 `SplitPaneHorizontalLayout`（209，左右分屏），但它是通用的分屏容器，缺少**专门面向数据对比场景的差异化布局模板**。对比布局有其特殊需求：差异高亮、同步滚动、行号对齐、以及"相同/不同/仅左侧/仅右侧"的筛选控制。

本功能提供三种专门的对比布局模板，让开发者可以快速搭建专业的数据对比界面。

## 功能描述

包含以下三种数据对比布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `SideBySideComparisonLayout` | 左右并列对比：两栏同步滚动，差异行高亮 | 文档版本对比、配置对比 |
| `InlineDiffLayout` | 行内差异：在同一行展示旧值和新值，删除线+新值高亮 | 短文本对比、字段变更记录 |
| `TableRowComparisonLayout` | 表格行对比：左右两表格，差异单元格高亮，支持筛选 | 数据表对比、报表核对 |

每种布局均内建：
- 差异高亮（新增为绿色、删除为红色、修改为黄色/橙色）
- 同步滚动（两侧滚动位置保持一致）
- 差异筛选（仅看差异 / 仅看相同 / 全部）
- 差异计数（"共 12 处差异"）
- 与 `ScrollArea`、`Table`、`Badge` 的衔接

## 目标用户

- 需要实现版本对比、文件差异、数据核对的开发者
- 需要展示变更记录、审计追踪的开发者

## 详细设计

### 布局结构描述

#### 1. 左右并列对比（SideBySideComparisonLayout）

```
+--------------------------------------------------+
| 版本对比                              [仅差异 ▼]  |
+--------------------------------------------------+
| 版本 A (昨天)              | 版本 B (今天)         |
| ┌────────────────────────┐ ┌────────────────────┐|
| │ 1  项目名称: Demo      │ │ 1  项目名称: Demo  │|
| │ 2  状态: 进行中        │ │ 2  状态: 已完成    ││
| │    ─────────────────   │ │    +++++++++++++++  ││
| │ 3  负责人: 张三        │ │ 3  负责人: 李四    ││
| │    ─────────────────   │ │    +++++++++++++++  ││
| │ 4  截止: 2024-06-30    │ │ 4  截止: 2024-07-15││
| │    +++++++++++++++++   │ │    +++++++++++++++  ││
| │ 5  描述: ...           │ │ 5  描述: ...       │|
| └────────────────────────┘ └────────────────────┘|
| 共 3 处差异                      [上一个] [下一个]│
+--------------------------------------------------+
| ↑ 差异行背景高亮，同步滚动                         |
+--------------------------------------------------+
```

#### 2. 行内差异（InlineDiffLayout）

```
| ┌────────────────────────────────────────────┐  |
| │ 状态                                          │  |

| ├────────────────────────────────────────────┤  |
| │ 负责人                                        │  |
| │   ~~张三~~ → **李四**                         │  |
| ├────────────────────────────────────────────┤  |
| │ 截止日期                                      │  |
| │   ~~2024-06-30~~ → **2024-07-15**             │  |
| └────────────────────────────────────────────┘  |
|                                                  |
| 2024-06-10 09:00  李四                           |
| ┌────────────────────────────────────────────┐  |
| │ 描述                                          │  |

```

#### 3. 表格行对比（TableRowComparisonLayout）

```
+--------------------------------------------------+
| 数据核对                              [仅差异 ▼]  |
+--------------------------------------------------+
| ┌────────────────────────────────────────────┐  |
| │ 字段    │ 数据源 A   │ 数据源 B   │ 状态    │  |
| ├────────────────────────────────────────────┤  |
| │ 名称    │ 产品 A     │ 产品 A     │ ✓ 一致  │  |
| │ 价格    │ ¥100      │ ¥120      │ ✗ 差异  │  |
| │ 库存    │ 50        │ 50         │ ✓ 一致  │  |
| │ 分类    │ 电子      │ 数码       │ ✗ 差异  │  |
| │ 上架    │ 是        │ 是         │ ✓ 一致  │  |
| └────────────────────────────────────────────┘  |
| 共 5 行，2 处差异                                |
+--------------------------------------------------+
```

### 涉及的技术点

- 使用 CSS Grid 或 Table 实现对比内容的严格列对齐
- 使用 `ResizablePanelGroup` 实现左右分栏的拖拽调宽
- 使用 `ScrollArea` 包裹对比内容，通过 `syncScroll` 保持两侧滚动同步
- 使用 `Badge` 展示差异状态（一致/差异/新增/删除）
- 使用主题 token 定义差异颜色：`destructive`（删除）、`green-500`（新增）、`orange-500`（修改）
- 使用 `ButtonGroup` 实现差异筛选切换（全部/仅差异/仅相同）

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/SideBySideComparisonLayout.vue`
- 新增 `src/components/layouts/InlineDiffLayout.vue`
- 新增 `src/components/layouts/TableRowComparisonLayout.vue`

**Props 设计（以 TableRowComparisonLayout 为例）：**

```ts
interface ComparisonRow {
  field: string
  valueA: string | number
  valueB: string | number
  status: 'same' | 'different' | 'added' | 'removed'
}

interface TableRowComparisonLayoutProps {
  rows: ComparisonRow[]
  labelA?: string            // 左侧标签，如"数据源 A"
  labelB?: string            // 右侧标签，如"数据源 B"
  filter?: 'all' | 'different' | 'same'
  showStatusColumn?: boolean // 是否显示状态列，默认 true
}

// Events
// @filter-change — 筛选条件变化
// @row-click     — 点击某行
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `SideBySideComparisonLayout` 的两侧内容在垂直滚动时保持同步，误差不超过一行高度
- [ ] 差异高亮颜色在暗色模式下自动适配，保持足够的对比度
- [ ] `InlineDiffLayout` 的删除文本使用删除线（`line-through`），新增文本使用粗体高亮
- [ ] `TableRowComparisonLayout` 的差异行可通过筛选控制显示/隐藏，切换时无布局跳动
- [ ] 差异计数在筛选后自动更新（如"共 5 行，2 处差异" → 筛选后"共 2 行，2 处差异"）
- [ ] 支持通过 `syncScroll` prop 控制是否开启同步滚动
- [ ] 所有差异颜色使用 CSS 变量或主题 token，不硬编码色值
