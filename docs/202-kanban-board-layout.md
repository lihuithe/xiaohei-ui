# 202 看板布局模板（Kanban Board Layout）

## 功能名称

看板布局模板 — 提供多列拖拽式看板骨架，支持列头、卡片占位和水平滚动。

## 功能背景/动机

看板（Kanban）是任务管理、内容审核、订单流转等场景的经典布局模式。现有脚手架已具备 `Card`、`Badge`、`Avatar`、`DropdownMenu` 等原子组件，但缺少**页面级的看板布局编排**。开发者在实现看板时，需要自行处理列的水平排列、卡片在列内的垂直堆叠、以及滚动条行为，容易因实现不一致导致体验参差不齐。

本功能提供一个纯粹的**UI 布局模板**，不包含任何业务拖拽逻辑（如 `@vueuse/motion` 或 `sortablejs` 的绑定），只提供看板的结构骨架和样式规范。开发者可根据业务需要自行接入拖拽库。

## 功能描述

包含以下看板布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `KanbanBoardLayout` | 基础看板：多列水平排列，列内卡片垂直堆叠 | 通用任务看板、内容审核 |
| `KanbanBoardWithAddColumn` | 带"添加列"占位按钮的扩展看板 | 可动态扩展阶段的场景 |
| `KanbanCompactLayout` | 迷你看板：窄列宽、紧凑卡片、适合侧边嵌入 | 仪表盘中的小型看板预览 |

每种布局均内建：
- 列头区（标题 + 计数 Badge + 操作菜单占位）
- 列内容区的滚动处理（独立垂直滚动）
- 看板整体的水平滚动处理
- 卡片占位槽（接受任意卡片组件）
- 空列状态展示（与现有 `Empty` 组件衔接）

## 目标用户

- 需要实现任务管理、流程审批、内容生产流水线等看板视图的开发者
- 希望在仪表盘内嵌入小型看板预览的开发者

### 涉及的技术点

- 使用 `flex` + `overflow-x-auto` 实现看板整体水平滚动
- 每列内部使用 `flex-col` + `overflow-y-auto` 实现独立垂直滚动
- 使用现有 `Card`、`Badge`、`Empty`、`Button` 组件
- 列宽通过 CSS 变量或 props 控制，支持紧凑模式
- 可选：在列头使用 `DropdownMenu` 放置列操作菜单（重命名、删除、颜色标记）

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/KanbanBoardLayout.vue`
- 新增 `src/components/layouts/KanbanCompactLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 可与 `DashboardLayout` 组合（在仪表盘内嵌入 `KanbanCompactLayout`）

**Props 设计（以 KanbanBoardLayout 为例）：**

```ts
interface KanbanColumn {
  id: string
  title: string
  color?: string          // 列头底部装饰线颜色
}

interface KanbanBoardLayoutProps {
  columns: KanbanColumn[]
  columnWidth?: number     // 默认 280px
  gap?: number             // 列间距，默认 16px
  compact?: boolean        // 是否紧凑模式
  showAddColumn?: boolean  // 是否显示添加列按钮
}

// Slot 约定
// #column-header-{columnId}  — 自定义列头（可选）
// #column-{columnId}         — 列内容（卡片列表）
// #column-footer-{columnId}  — 列底部（如"添加卡片"按钮）
// #empty-{columnId}          — 空列状态（默认使用 Empty 组件）
```

## 验收标准

- [ ] 看板在列数溢出视口宽度时，整体可水平滚动，且滚动条样式符合主题
- [ ] 每列的卡片列表可独立垂直滚动，滚动不影响其他列
- [ ] 列头在垂直滚动时保持固定（sticky）
- [ ] 空列时自动展示 `Empty` 占位（可自定义 slot 覆盖）
- [ ] 迷你模式下列宽收缩至 180px，卡片内边距和字体同步缩小
- [ ] 所有 slot 命名遵循 `column-{id}` / `column-header-{id}` 约定
- [ ] 列头底部的颜色装饰线支持通过 `color` 字段自定义（符合主题 token）

## 优先级
