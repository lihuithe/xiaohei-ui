# 214 统计指标面板布局（Metric & Stat Cards Layout）

## 功能名称

统计指标面板布局 — 提供 KPI 仪表盘、对比指标组、趋势卡片三种数据指标展示骨架。

## 功能背景/动机

现有脚手架的 `DashboardLayout.vue` 已包含简单的统计卡片（4 个等宽卡片展示总用户、活跃用户等），但缺少**通用的、可复用的指标面板布局模板**。在实际桌面应用中，指标展示的需求非常多样：不同尺寸的重点指标（KPI）、带趋势线的对比卡片、网格化的指标矩阵等。开发者每次都需要重新设计卡片的尺寸比例、排列逻辑和响应式策略。

本功能提供三种专门的指标展示布局，覆盖从"一眼看清核心数据"到"多维度对比分析"的完整场景。

## 功能描述

包含以下三种指标面板布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `KpiCardsLayout` | 大字号 KPI 卡片，支持 1-4 个重点指标的突出展示 | 首页仪表盘、管理后台概览 |
| `MetricComparisonLayout` | 对比指标组：当前值 vs 目标值，带进度条或环形图 | 销售目标、项目进度、预算消耗 |
| `MetricTrendLayout` | 趋势卡片：指标值 + 迷你折线/柱状图 + 同比环比 | 流量监控、收入增长、用户活跃 |

每种布局均内建：
- 响应式断点（大屏多列 → 小屏单列）
- 数值格式化占位（支持千分位、百分比、货币符号）
- 与 `ChartContainer`（已有图表组件）的衔接槽位
- 加载状态（`Skeleton` 数字占位）
- 点击卡片后的详情展开/跳转占位

## 目标用户

- 需要实现仪表盘、数据概览、监控面板的开发者
- 需要展示核心指标、进度、趋势的分析类应用开发者

### 涉及的技术点

- 使用现有 `Card`、`Badge`、`Progress` 组件构建卡片结构
- 使用现有 `ChartContainer`（`ui/chart/`）作为迷你图表的容器
- 使用 `ui/skeleton/Skeleton.vue` 作为数值加载占位
- 使用 CSS Grid 的 `grid-template-areas` 实现 KPI 的混合尺寸布局
- 使用 `lucide-vue-next` 的 `TrendingUp`、`TrendingDown` 图标表示涨跌

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/KpiCardsLayout.vue`
- 新增 `src/components/layouts/MetricComparisonLayout.vue`
- 新增 `src/components/layouts/MetricTrendLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 可与 `DashboardLayout` 组合（替换其内置的简单统计卡片区域）
- 可与 `CardGridLayout` 组合（将指标卡片嵌入更大的卡片网格中）

### 需要新增/修改的文件

```
desktop-app/src/components/layouts/
  ├── KpiCardsLayout.vue            # KPI 重点指标卡片
  ├── MetricComparisonLayout.vue    # 对比指标组
  ├── MetricTrendLayout.vue         # 趋势卡片
  └── index.ts                      # 追加导出
```

**Props 设计（以 KpiCardsLayout 为例）：**

```ts
interface KpiItem {
  label: string
  value: string | number
  change?: number           // 变化率，正数为涨，负数为跌
  changeLabel?: string      // 如"同比"、"环比"
  size?: 'lg' | 'md' | 'sm' // 卡片尺寸，默认 md
  icon?: Component
}

interface KpiCardsLayoutProps {
  items: KpiItem[]
  loading?: boolean
  columns?: {              // 响应式列数
    sm?: number
    md?: number
    lg?: number
  }
}
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] KPI 卡片的数值加载时显示 `Skeleton` 占位，宽度与预期数值长度一致
- [ ] 对比指标组的进度条颜色随完成度变化：小于 50% 为默认色，50-80% 为警告色，大于 80% 为成功色
- [ ] 趋势卡片的迷你图表支持通过 `#chart` slot 接入任意图表库
- [ ] 响应式：大屏 4 列 → 中屏 2 列 → 小屏 1 列，无横向滚动
- [ ] 涨跌指示器使用 `TrendingUp`（绿色）和 `TrendingDown`（红色）图标，颜色使用主题 token
- [ ] 点击卡片触发 `click` 事件，可跳转详情页或展开抽屉

## 优先级

P1 — 指标面板是仪表盘类应用的核心，与已有的 `DashboardLayout` 形成升级替代关系。
