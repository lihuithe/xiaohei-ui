# 212 加载骨架屏布局（Skeleton Screen Layout）

## 功能名称

加载骨架屏布局 — 提供列表骨架、卡片网格骨架、详情页骨架、表格骨架四种页面级骨架屏模板。

## 功能背景/动机

现有脚手架已有 `ui/skeleton/Skeleton.vue`（一个基础的脉冲动画矩形）和 `105-loading-empty-error-state-system`（加载状态的统一设计规范），但缺少**面向不同页面结构的骨架屏布局模板**。开发者在实现加载状态时，往往需要为列表、表格、详情页、仪表盘等不同页面手动拼凑多个 `Skeleton` 组件，位置和比例难以与真实内容对齐，导致加载完成后页面产生明显的布局跳动（Layout Shift）。

本功能提供与真实页面结构 1:1 对应的骨架屏布局，让开发者在数据加载期间展示"内容的轮廓"，提升感知性能并减少布局跳动。

## 功能描述

包含以下四种骨架屏布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `SkeletonListLayout` | 列表骨架：带头像/图标、标题行、描述行的重复项 | 消息列表、通知列表、搜索建议 |
| `SkeletonCardGridLayout` | 卡片网格骨架：图片区 + 标题 + 描述 + 操作按钮 | 资源库、项目卡片、相册加载 |
| `SkeletonDetailLayout` | 详情页骨架：左侧摘要 + 右侧内容区 + 属性列表 | 人员详情、订单详情、文件属性 |
| `SkeletonTableLayout` | 表格骨架：表头行 + 多行数据行（每行多列） | 数据表格、日志列表、用户管理 |

每种布局均内建：
- 与对应真实布局的 DOM 结构对齐（相同的 margin、padding、gap）
- 可调节的骨架项数量（通过 props）
- 脉冲动画（与现有 `Skeleton` 组件一致）
- 与 `Empty`、`Error` 状态的无缝切换（通过 `v-if` 控制同一位置）

## 目标用户

- 需要在数据加载期间提供流畅过渡体验的开发者
- 希望减少页面加载完成后布局跳动的开发者

### 涉及的技术点

- 使用现有 `ui/skeleton/Skeleton.vue` 作为基础图形元素
- 使用 `shadcn-vue` 的 `Skeleton` 组合模式（圆角、高度、宽度通过 class 控制）
- 使用 CSS Grid / Flex 构建骨架布局，确保与真实页面完全对齐
- 使用 `animate-pulse` 动画（已有）
- 使用 `shimmer` 动画作为可选增强（从左到右的光带扫过效果）

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/SkeletonListLayout.vue`
- 新增 `src/components/layouts/SkeletonCardGridLayout.vue`
- 新增 `src/components/layouts/SkeletonDetailLayout.vue`
- 新增 `src/components/layouts/SkeletonTableLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 各骨架布局与对应的真实布局（`DataTableLayout`、`CardGridLayout`、`DetailSplitLayout`）的容器尺寸完全一致，可直接在同一位置 `v-if` 切换

**Props 设计（以 SkeletonListLayout 为例）：**

```ts
interface SkeletonListLayoutProps {
  rows?: number          // 骨架行数，默认 5
  showAvatar?: boolean   // 是否显示头像占位，默认 true
  showDescription?: boolean // 是否显示描述行，默认 true
  avatarShape?: 'circle' | 'square'  // 头像形状，默认 circle
}
```

## 验收标准

- [ ] 四种骨架屏布局均能在 `ComponentPlayground` 中独立预览
- [ ] 骨架屏的外边距、内边距、间距与对应的真实页面布局完全一致
- [ ] 骨架项的圆角（`rounded-md`、`rounded-full`）与真实组件一致
- [ ] 支持可选的 `shimmer` 动画模式（从左到右的光带效果）
- [ ] 支持通过 `rows`/`columns` props 动态调整骨架数量
- [ ] 在数据加载完成后，骨架屏与真实内容的切换无视觉跳动
- [ ] 暗色模式下骨架颜色自动适配（使用 `bg-muted` token）

## 优先级

P1 — 骨架屏是感知性能和用户体验的重要组成部分，与已有的 `105` 和 `Skeleton` 组件形成完整体系。
