# 211 表单内联编辑模式（Inline Edit Form Layout）

## 功能名称

表单内联编辑模式 — 提供单元格内联编辑、行内编辑、卡片内编辑三种可复用骨架。

## 功能背景/动机

在桌面应用中，"内联编辑"是提升操作效率的关键交互模式：用户无需跳转到独立表单页面，直接在数据展示位置完成修改。现有脚手架已有丰富的表单原子组件（`Input`、`Field`、`FormItem` 等）和表格组件（`Table`），但缺少**内联编辑的布局编排模板**。开发者在实现内联编辑时，需要自行处理"展示态 → 编辑态"的切换动画、取消/保存的键盘快捷键、以及编辑框与周围元素的布局对齐。

本功能提供三种内联编辑的布局模板，覆盖表格、列表、卡片三大数据展示场景，确保编辑体验的一致性和可复用性。

## 功能描述

包含以下三种内联编辑布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `InlineCellEditLayout` | 表格单元格内直接切换为输入框，失焦或回车保存 | 数据表格的快速修正、电子表格风格 |
| `InlineRowEditLayout` | 整行进入编辑模式，所有字段变为可编辑，行尾出现保存/取消 | 人员信息列表、库存管理 |
| `InlineCardEditLayout` | 卡片内部字段点击后进入编辑，卡片整体出现编辑边框 | 项目卡片、个人资料卡片 |

每种布局均内建：
- 展示态与编辑态的平滑切换动画
- 保存/取消的键盘支持（Enter 保存、Esc 取消）
- 验证错误提示（与 `FormMessage` 组件衔接）
- 加载状态（保存中的禁用和 loading 指示）
- 批量编辑模式（多行同时编辑，统一保存）

## 目标用户

- 需要实现表格、列表、卡片的就地编辑功能的开发者
- 希望减少页面跳转、提升数据修改效率的场景

## 详细设计

### 布局结构描述

### 涉及的技术点

- 使用现有 `Input`、`Select`、`Field`、`FormMessage` 组件作为编辑态控件
- 使用 `v-if/v-else` 或动态组件切换展示态与编辑态
- 使用 `nextTick` + `focus()` 确保编辑态激活时自动聚焦输入框
- 使用 `AnimatedTransition` 实现切换时的淡入淡出动画
- 使用 `Pinia` 或本地状态管理编辑行的草稿数据（支持取消回滚）

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/InlineCellEditLayout.vue`
- 新增 `src/components/layouts/InlineRowEditLayout.vue`
- 新增 `src/components/layouts/InlineCardEditLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 可与 `DataTableLayout` 组合（表格内启用行编辑）
- 可与 `CardGridLayout` 组合（卡片内启用字段编辑）

### 需要新增/修改的文件


```ts
  }[]
  saving?: boolean
}

// Events
// @save   — 用户点击保存或按 Enter
// @cancel — 用户点击取消或按 Esc
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] 点击展示态区域后，编辑态在 150ms 内完成切换并自动聚焦
- [ ] 按 Enter 触发保存，按 Esc 触发取消并恢复原始值
- [ ] 保存中时，保存按钮显示 loading 状态，输入框禁用编辑
- [ ] 验证失败时，错误提示出现在对应字段下方，与 `FormMessage` 样式一致
- [ ] 批量编辑模式下，支持多行同时编辑，底部出现统一的"保存全部/取消全部"操作栏
- [ ] 卡片编辑模式下，卡片边框在进入编辑态时有 primary 色高亮动画

## 优先级

P1 — 内联编辑是生产力应用的高频交互，能显著提升用户体验。
