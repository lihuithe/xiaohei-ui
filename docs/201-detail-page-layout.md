# 201 详情页布局模板（Detail Page Layout）

## 功能名称

详情页布局模板 — 提供左右分栏、上下结构、可折叠区块三种详情展示骨架。

## 功能背景/动机

桌面应用中"详情页"是仅次于列表/表格的高频场景：用户从列表选中一条记录后，需要查看完整信息、关联数据、操作历史等。现有脚手架已有 `DataTableLayout`（列表侧），但缺少与其配套的**详情展示侧布局模板**。开发者在实现详情页时往往各自为战，导致信息层级混乱、阅读效率低下。

本功能提供一套结构化的详情页布局，规范信息展示层级（摘要 → 属性 → 关联 → 操作），确保不同业务详情页的一致性体验。

## 功能描述

包含以下三种详情页布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `DetailSplitLayout` | 左侧信息面板 + 右侧详情内容，支持拖拽调整宽度 | 人员详情、订单详情、文件属性 |
| `DetailStackedLayout` | 顶部摘要卡片 + 下方 Tab 内容区 | 项目详情、文章详情、报表详情 |
| `DetailAccordionLayout` | 全宽页面，各区块以手风琴形式折叠 | 系统日志、审计详情、长文档 |

每种布局均内建：
- 面包屑/返回导航区
- 操作按钮区（编辑、删除、导出、打印占位）
- 元信息展示模式（标签-值对齐列表）
- 与 `DataTableLayout` 组合为"主从布局"的衔接约定

## 目标用户

- 需要实现"列表 → 详情"跳转流的开发者
- 需要展示复杂对象多维度信息的场景

## 详细设计

### 布局结构描述

#### 1. 左右分栏（DetailSplitLayout）

```
+----------------------------------------------------------+
| ← 返回  订单详情 #1024                         [编辑][删除]|
+----------------------------------------------------------+
| 左侧面板 (260px, 可拖拽)  |  右侧详情内容区              |
| ┌─────────────────────┐  |  ┌─────────────────────────┐ |
| │ 摘要卡片            │  |  │ Tabs: [概览][物流][日志]│ |
| │ 状态: 已完成        │  |  ├─────────────────────────┤ |
| │ 金额: ¥2,400        │  |  │                         │ |
| │ 客户: 张三          │  |  │   当前 Tab 内容           │ |
| └─────────────────────┘  |  │                         │ |
| ┌─────────────────────┐  |  │                         │ |
| │ 属性列表            │  |  └─────────────────────────┘ |
| │ 创建时间  2024-...  │  |                              |
| │ 支付方式  支付宝    │  |                              |
| │ ...                 │  |                              |
| └─────────────────────┘  |                              |
+----------------------------------------------------------+
        ↑ ResizableHandle
```

#### 2. 上下结构（DetailStackedLayout）

```
+--------------------------------------------------+
| ← 返回  项目名称                                |
+--------------------------------------------------+
| ┌──────────────────────────────────────────────┐ |
| │  [图标]  项目代号: PROJ-001                   │ |
| │  描述: 这是一个示例项目描述...                 │ |
| │  标签: [设计中] [进行中]        [编辑][归档]   │ |
| └──────────────────────────────────────────────┘ |
| Tabs: [任务][成员][文件][活动]                    |
+--------------------------------------------------+
|                                                  |
|  当前 Tab 内容区（表格/列表/时间线）              |
|                                                  |
+--------------------------------------------------+
```

#### 3. 可折叠区块（DetailAccordionLayout）

```
+--------------------------------------------------+
| ← 返回  系统审计日志 #Audit-2024-001             |
+--------------------------------------------------+
| ▼ 基础信息                                       |
|   请求ID / 用户 / IP / 时间                      |
| ▶ 请求参数（点击展开）                           |
| ▶ 响应结果（点击展开）                           |
| ▼ 调用链路                                       |
|   服务A → 服务B → 数据库（时间线组件）           |
| ▶ 异常堆栈（点击展开，红色高亮）                 |
+--------------------------------------------------+
```

### 涉及的技术点

- 使用现有 `ResizablePanelGroup`、`ResizablePanel`、`ResizableHandle` 实现左右分栏的拖拽调整
- 使用现有 `Tabs`、`Card`、`Accordion`、`Badge`、`Avatar` 构建信息区块
- 使用现有 `Breadcrumb` 或 `Item` 组件构建顶部导航
- 使用 `ScrollArea` 确保左右面板独立滚动

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/DetailSplitLayout.vue`
- 新增 `src/components/layouts/DetailStackedLayout.vue`
- 新增 `src/components/layouts/DetailAccordionLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 可与 `DataTableLayout` 组合为经典"左列表右详情"桌面应用主从布局（通过 Router 或 props 控制）

### 需要新增/修改的文件

```
desktop-app/src/components/layouts/
  ├── DetailSplitLayout.vue       # 左右分栏详情
  ├── DetailStackedLayout.vue     # 上下结构详情
  ├── DetailAccordionLayout.vue   # 可折叠区块详情
  └── index.ts                    # 追加导出
```

**Props 设计（以 DetailSplitLayout 为例）：**

```ts
interface DetailSplitLayoutProps {
  defaultLeftWidth?: number        // 默认左侧面板宽度，默认 260
  minLeftWidth?: number            // 最小宽度，默认 200
  maxLeftWidth?: number            // 最大宽度，默认 400
  collapsible?: boolean            // 左侧面板是否可完全收起
}

// Slot 约定
// #header       — 顶部标题与操作按钮
// #left         — 左侧面板内容
// #default      — 右侧主内容区
// #left-footer  — 左侧面板底部（可选）
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `DetailSplitLayout` 的左侧面板支持鼠标拖拽调整宽度，有视觉拖拽指示器
- [ ] `DetailSplitLayout` 左侧面板收起后，主内容区自动填满剩余空间
- [ ] `DetailStackedLayout` 的 Tab 切换保留滚动位置（或提供配置项）
- [ ] `DetailAccordionLayout` 支持同时展开多个区块（或仅展开一个，通过 props 控制）
- [ ] 所有布局的顶部操作区支持 `#header-actions` slot 自定义
- [ ] 与 `DataTableLayout` 的配色和圆角风格保持一致

## 优先级

P0 — 详情页是桌面应用数据浏览的核心环节，与已有 `DataTableLayout` 形成完整闭环。
