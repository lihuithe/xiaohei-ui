# 233 设置分类布局（Settings Category Layout）

## 功能名称

设置分类布局 — 提供卡片式设置、搜索驱动设置、分组折叠设置三种可复用骨架。

## 功能背景/动机

现有脚手架已有 `005-settings-configuration-framework`（设置页配置管理框架）和 `SettingsLayout.vue`（左侧标签导航 + 右侧内容区的设置页），但 `SettingsLayout` 仅提供一种经典的标签页式设置布局。在桌面应用中，设置页面的形态非常多样：macOS 风格的图标卡片式设置、VS Code 风格的搜索驱动设置、以及大量字段时的分组折叠布局。

本功能在 `005` 和 `SettingsLayout` 基础上，提供三种设置页的布局变体，让开发者可以根据设置的复杂度和数量选择最合适的展示方式。

## 功能描述

包含以下三种设置分类布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `CardSettingsLayout` | 卡片式设置：大图标分类卡片网格，点击进入子设置页 | 设置项较少、分类明确的应用 |
| `SearchDrivenSettingsLayout` | 搜索驱动设置：顶部搜索框 + 实时过滤的设置项列表 | 设置项极多、需要快速定位 |
| `GroupedSettingsLayout` | 分组折叠设置：所有设置项在一个长页面，按组折叠 | 设置项中等、喜欢一次性概览 |

每种布局均内建：
- 与 `005-settings-configuration-framework` 的配置项数据结构衔接
- 修改状态提示（"有未保存更改"指示器）
- 重置/恢复默认值按钮占位
- 与 `Field`、`FormItem`、`Switch`、`Select` 的无缝兼容

## 目标用户

- 需要实现设置/偏好/配置页面的开发者
- 希望根据设置项数量选择最佳展示方式的开发者

### 涉及的技术点

- 使用现有 `Card`、`Input`、`Switch`、`Select`、`Field`、`FormItem` 组件
- 使用 `ui/accordion/` 实现分组折叠
- 使用 `SearchBar`（已有）或 `Command`（已有）实现搜索驱动设置的过滤
- 使用 `Badge` 展示有未保存更改的分组
- 使用 `Button` 构建保存/取消/重置操作栏
- 卡片式布局使用 CSS Grid 的响应式列数（1→2→3 列）

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/CardSettingsLayout.vue`
- 新增 `src/components/layouts/SearchDrivenSettingsLayout.vue`
- 新增 `src/components/layouts/GroupedSettingsLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 三种布局均可替代现有的 `SettingsLayout.vue`，根据场景选择
- `GroupedSettingsLayout` 可与 `FormSingleColumnLayout`（200）组合（分组内的表单字段布局）

### 需要新增/修改的文件

```
desktop-app/src/components/layouts/
  ├── CardSettingsLayout.vue        # 卡片式设置
  ├── SearchDrivenSettingsLayout.vue # 搜索驱动设置
  ├── GroupedSettingsLayout.vue     # 分组折叠设置
  └── index.ts                      # 追加导出
```

**Props 设计（以 CardSettingsLayout 为例）：**

```ts
interface SettingsCategory {
  id: string
  title: string
  description?: string
  icon: Component
  badge?: string           // 如"有更新"、"新"
}

interface CardSettingsLayoutProps {
  categories: SettingsCategory[]
  columns?: { sm?: number; md?: number; lg?: number }
}

// Events
// @category-click — 点击分类卡片
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `CardSettingsLayout` 的卡片在窗口变化时自动重排：大屏 3 列 → 中屏 2 列 → 小屏 1 列
- [ ] `SearchDrivenSettingsLayout` 的搜索框支持实时过滤，输入后 150ms 内展示结果
- [ ] `SearchDrivenSettingsLayout` 的高亮匹配关键词（如将"主题"加粗显示）
- [ ] `GroupedSettingsLayout` 的分组展开/收起有高度过渡动画
- [ ] 有未保存更改时，保存按钮从禁用变为可用，分组标题旁显示圆点指示
- [ ] 点击"重置"后弹出确认对话框（使用现有 `AlertDialog`）
- [ ] 所有设置字段与 `Field`、`FormItem` 的错误提示样式无缝兼容

## 优先级

P1 — 设置页是桌面应用的标准页面，与已有的 `005` 和 `SettingsLayout` 形成丰富变体。
