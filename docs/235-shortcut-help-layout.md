# 235 快捷键帮助页布局（Shortcut Help Layout）

## 功能名称

快捷键帮助页布局 — 提供快捷键速查表、交互式快捷键映射、快捷键搜索三种可复用骨架。

## 功能背景/动机

现有脚手架已有 `001-global-shortcuts-system`（全局快捷键系统）和 `ui/kbd/`（键盘按键组件族：`Kbd`、`KbdGroup`），但缺少**面向用户的快捷键帮助/速查页面布局模板**。在桌面应用中，快捷键是提升效率的核心手段，但用户往往难以记忆所有快捷键。一个结构清晰的快捷键帮助页可以显著降低学习成本。

本功能提供三种快捷键展示的页面布局，与 `001` 的快捷键数据和 `ui/kbd/` 的视觉规范衔接，让开发者可以快速生成交互式的快捷键参考页面。

## 功能描述

包含以下三种快捷键帮助布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `ShortcutCheatsheetLayout` | 快捷键速查表：按功能分组，网格排列快捷键卡片 | 帮助中心、快捷键参考页 |
| `InteractiveShortcutMapLayout` | 交互式快捷键映射：虚拟键盘高亮 + 点击按键查看功能 | 快捷键教学、新手引导 |
| `ShortcutSearchLayout` | 快捷键搜索：顶部搜索框 + 实时过滤的快捷键列表 | 快捷键极多时的快速定位 |

每种布局均内建：
- 与 `ui/kbd/` 组件的视觉风格一致（按键样式、组合键展示）
- 按功能分组（文件、编辑、视图、导航等）
- 平台适配（macOS 显示 `⌘`，Windows 显示 `Ctrl`）
- 与 `Command`（已有命令面板）的衔接（搜索快捷键时可直接执行）

## 目标用户

- 需要为应用提供快捷键帮助页面的开发者
- 希望降低用户快捷键学习成本的开发者

### 涉及的技术点

- 使用现有 `ui/kbd/` 组件族（`Kbd`、`KbdGroup`）展示快捷键
- 使用 `Input`、`Command` 组件实现搜索功能
- 使用 `Card` 或 `Table` 构建速查表网格
- 使用 CSS Grid 实现虚拟键盘的按键排列
- 使用 `ScrollArea` 处理长列表的滚动
- 平台检测通过 `navigator.platform` 或已有的 `window.electronAPI.platform` 实现

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/ShortcutCheatsheetLayout.vue`
- 新增 `src/components/layouts/InteractiveShortcutMapLayout.vue`
- 新增 `src/components/layouts/ShortcutSearchLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 可与 `DocumentReaderLayout`（219）组合（快捷键帮助作为文档页面）
- 可与 `Command`（006 命令面板）组合（搜索快捷键时可直接触发命令）

```ts
interface ShortcutItem {
  id: string

interface ShortcutCheatsheetLayoutProps {
  shortcuts: ShortcutItem[]
  platform?: 'mac' | 'windows' | 'linux' | 'auto'

}

// Events
// @shortcut-click — 点击某个快捷键项
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `ShortcutCheatsheetLayout` 的按键组合使用 `ui/kbd/` 组件展示，视觉风格一致
- [ ] 平台自动检测：macOS 显示 `⌘` / `⌥` / `⌃`，Windows/Linux 显示 `Ctrl` / `Alt`
- [ ] `InteractiveShortcutMapLayout` 的虚拟键盘按键在点击后有按下效果（背景色加深）
- [ ] `ShortcutSearchLayout` 支持实时过滤，输入后 100ms 内展示结果
- [ ] 搜索结果支持通过 Enter 键直接执行第一个匹配的快捷键功能
- [ ] 快捷键列表支持通过分类标签快速筛选
- [ ] 暗色模式下 `Kbd` 组件的边框和背景色自动适配

## 优先级

P1 — 快捷键帮助页是生产力桌面应用的标准配置，与已有的 `001` 和 `ui/kbd/` 形成完整闭环。
