# 230 响应式布局变体（Responsive Layout Variants）

## 功能名称

响应式布局变体 — 提供紧凑型布局、平板适配布局、触控优化布局三种面向不同设备和窗口尺寸的骨架。

## 功能背景/动机

现有脚手架已有 `107-responsive-window-layout`（响应式窗口自适应布局的功能设计），它定义了断点策略和窗口尺寸变化时的行为准则。但缺少**具体的、可复用的响应式布局模板**。在桌面应用中，用户可能会将窗口缩得很小（分屏使用）、在平板模式下使用（触控交互）、或者在触控屏设备上操作（更大的点击区域）。开发者需要为同一页面准备多种布局变体，以适配不同的使用场景。

本功能提供三种专门的响应式布局变体，与 `107` 的设计规范衔接，将响应式策略落地为可直接使用的 Layout 组件。

## 功能描述

包含以下三种响应式布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `CompactLayout` | 紧凑型：窗口宽度小于 800px 时的简化布局，侧边栏收起为图标，内容区全宽 | 小窗口、分屏使用、侧边常驻其他应用 |
| `TabletLayout` | 平板适配：针对触控优化的布局，更大的按钮和间距，底部导航替代侧边栏 | 二合一设备、平板模式、触控屏 |
| `TouchOptimizedLayout` | 触控优化：所有可点击元素的最小尺寸为 44x44px，支持滑动手势占位 | 触控显示器、Kiosk 模式、演示设备 |

每种布局均内建：
- 与 `107-responsive-window-layout` 断点策略的衔接（sm: 640px, md: 768px, lg: 1024px, xl: 1280px）
- 布局切换时的过渡动画
- 触控/鼠标交互的差异化处理（hover 效果在触控设备上的降级）
- 与 `Sidebar`、`BottomNavLayout`（205）、`Sheet` 的衔接

## 目标用户

- 需要支持多种窗口尺寸和输入方式的桌面应用开发者
- 希望应用在不同设备（桌面、平板、触控屏）上均有良好体验的开发者

### 涉及的技术点

- 使用 Tailwind CSS 的响应式前缀（`sm:`、`md:`、`lg:`）实现断点适配
- 使用 `window.matchMedia` 或 `useBreakpoints`（VueUse）检测当前断点
- 使用 CSS 变量控制触控模式下的间距和尺寸（`--touch-spacing`、`--touch-min-size`）
- 使用 `ui/sheet/` 在紧凑模式下将侧边栏转为底部 sheet 或侧边 drawer
- 使用 `BottomNavLayout`（205）作为平板模式的底部导航
- 使用 CSS `touch-action: pan-y` 为可滑动列表项配置触控行为

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/CompactLayout.vue`
- 新增 `src/components/layouts/TabletLayout.vue`

**Props 设计（以 TouchOptimizedLayout 为例）：**

```ts
interface TouchOptimizedLayoutProps {
  minTouchSize?: number     // 最小触控尺寸(px)，默认 44
  listItemHeight?: number    // 列表项高度(px)，默认 64
  buttonHeight?: number      // 按钮高度(px)，默认 48
  enableSwipeActions?: boolean // 是否启用列表项滑动手势，默认 true
  hideHoverEffects?: boolean   // 是否隐藏 hover 效果（触控设备），默认 true
}

// Slot 约定
// #default      — 主内容区
// #navigation   — 导航区（侧边栏或底部导航）
// #header       — 顶部标题栏
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `CompactLayout` 在窗口宽度小于 800px 时自动简化：侧边栏收起、卡片网格列数减少、表格横向滚动
- [ ] `TabletLayout` 的底部导航栏在窗口高度小于 600px 时自动隐藏，最大化内容区
- [ ] `TouchOptimizedLayout` 的所有可点击元素的最小尺寸不小于 44x44px
- [ ] `TouchOptimizedLayout` 的列表项支持左滑/右滑显示操作按钮（至少提供占位 slot）
- [ ] 布局切换时有 200ms 的过渡动画，无视觉跳动
- [ ] hover 效果在触控设备上自动降级（不显示 hover 状态，避免 sticky hover）
- [ ] 与 `107-responsive-window-layout` 的断点定义保持一致

## 优先级

P1 — 响应式适配是桌面应用跨设备体验的基础，与已有的 `107` 形成规范到落地的闭环。
