# 216 富文本编辑器占位模板（Rich Text Editor Layout）

## 功能名称

富文本编辑器占位模板 — 提供经典工具栏编辑器、极简分屏编辑器、便签式编辑器三种编辑器页面骨架。

## 功能背景/动机

桌面应用中大量场景需要文本编辑功能：笔记、文档、评论、邮件草稿等。现有脚手架已有 `Textarea` 组件，但缺少**面向富文本编辑场景的页面级布局模板**。开发者在集成第三方编辑器（如 TipTap、Quill、Monaco Editor 等）时，需要自行处理工具栏位置、编辑器容器自适应、字数统计、保存状态提示等通用结构。

本功能提供纯粹的 UI 布局骨架，不绑定任何具体的编辑器实现，只提供编辑器的"外壳"布局，让开发者可以即插即用地接入任意编辑引擎。

## 功能描述

包含以下三种编辑器布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `ClassicEditorLayout` | 顶部固定工具栏 + 编辑区 + 底部状态栏 | 通用文档编辑、富文本内容创作 |
| `SplitEditorLayout` | 左侧源码/编辑区 + 右侧实时预览，中间可拖拽 | Markdown、HTML、代码文档编辑 |
| `StickyNoteEditorLayout` | 无边框便签风格，标题 + 内容 + 标签，自动保存 | 快速笔记、备忘录、草稿箱 |

每种布局均内建：
- 工具栏占位槽（接受任意工具栏组件）
- 编辑器主区域的自适应高度（占据剩余视口空间）
- 底部状态栏（字数、保存状态、最后编辑时间占位）
- 标题输入区（可折叠或始终展示）
- 暗色/亮色主题适配

## 目标用户

- 需要集成富文本编辑器、Markdown 编辑器、代码编辑器的开发者
- 需要实现笔记、文档、评论编辑页面的开发者

## 详细设计

### 布局结构描述

#### 1. 经典编辑器（ClassicEditorLayout）

```
+--------------------------------------------------+
| [文档标题...                    ]        [保存]  |
+--------------------------------------------------+
| 工具栏占位槽                                      |
| [B] [I] [U] [H1] [H2] [🔗] [🖼️] [◦] [1.]     |
+--------------------------------------------------+
|                                                  |
|  编辑器主内容区                                   |
|  （自适应高度，占满剩余视口空间）                  |
|                                                  |
|  这里接入具体的编辑器组件...                      |
|                                                  |
|                                                  |
|                                                  |
+--------------------------------------------------+
| 字数: 1,240  |  已保存 14:32  |  [导出 ▼]        |
+--------------------------------------------------+
```

#### 2. 分屏编辑器（SplitEditorLayout）

```
+--------------------------------------------------+
| [文档标题...                    ]        [保存]  |
+--------------------------------------------------+
| 工具栏占位槽                                      |
+--------------------------------------------------+
| 编辑区 (50%)    ||    预览区 (50%)                |
| +-------------+  ||  +-------------+              |
| | # 标题      |  ||  | <h1>标题</h1>|              |
| |             |  ||  |             |              |
| | 内容...     |  ||  | 渲染内容... |              |
| |             |  ||  |             |              |
| +-------------+  ||  +-------------+              |
|     ↑ ResizableHandle                            |
| [同步滚动 ✓]                     [预览模式 ▼]    |
+--------------------------------------------------+
```

#### 3. 便签式编辑器（StickyNoteEditorLayout）

```
+--------------------------------------------------+
|                                                  |
|   ┌──────────────────────────────────────┐      |
|   │ [笔记标题...            ]  [标签 ▼]   │      |
|   ├──────────────────────────────────────┤      |
|   │                                      │      |
|   │  快速记录你的想法...                  │      |
|   │                                      │      |
|   │                                      │      |
|   │                                      │      |
|   ├──────────────────────────────────────┤      |
|   │  正在保存...                  14:32   │      |
|   └──────────────────────────────────────┘      |
|                                                  |
|   其他笔记卡片（网格排列）...                     |
|   ┌──────┐ ┌──────┐ ┌──────┐                     |
|   │ 笔记2│ │ 笔记3│ │ 笔记4│                     |
|   └──────┘ └──────┘ └──────┘                     |
|                                                  |
+--------------------------------------------------+
```

### 涉及的技术点

- 使用 `flex-col` + `flex-1` 实现编辑器区域自适应占满剩余空间
- 使用 `ui/resizable/` 实现 `SplitEditorLayout` 的拖拽分屏
- 使用 `ScrollArea` 包裹编辑器内容区，确保独立滚动
- 使用 `Input` 或原生标题输入框作为文档标题区
- 底部状态栏使用 `flex` + `gap` 排列状态项
- 便签式布局使用 CSS Grid 或 Flex Wrap 实现卡片网格

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/ClassicEditorLayout.vue`
- 新增 `src/components/layouts/SplitEditorLayout.vue`

**Props 设计（以 ClassicEditorLayout 为例）：**

```ts
interface ClassicEditorLayoutProps {
  title?: string
  showTitle?: boolean
  showToolbar?: boolean
  showStatusBar?: boolean
  statusBarItems?: {
    label: string
    value: string
  }[]
  autoSaveIndicator?: boolean  // 是否显示自动保存状态
}

// Slot 约定
// #toolbar      — 工具栏内容（富文本工具按钮）
// #editor       — 编辑器主组件（必填）
// #status-left  — 状态栏左侧额外内容
// #status-right — 状态栏右侧额外内容
// #title-extra  — 标题区右侧额外内容（如标签选择）
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] 编辑器主区域在窗口高度变化时自动占满剩余空间，无固定高度限制
- [ ] `ClassicEditorLayout` 的工具栏在垂直滚动时保持固定（sticky top）
- [ ] `SplitEditorLayout` 的拖拽手柄在 hover 时视觉加宽，拖拽时实时响应
- [ ] `StickyNoteEditorLayout` 的便签卡片支持通过 props 配置背景色（黄色/蓝色/绿色等便签色）
- [ ] 底部状态栏的保存状态支持三种视觉："已保存"（绿色圆点）、"保存中"（脉冲动画）、"有未保存更改"（黄色圆点）
- [ ] 暗色模式下编辑器区域与工具栏的边框对比度适配
- [ ] 所有布局不绑定具体编辑器实现，`#editor` slot 可放置任意 Vue 组件

## 优先级

P1 — 编辑器布局是生产力应用的核心场景，通用性强。
