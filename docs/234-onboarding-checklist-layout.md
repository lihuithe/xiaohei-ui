# 234 引导检查清单布局（Onboarding Checklist Layout）

## 功能名称

引导检查清单布局 — 提供新用户任务清单、功能发现引导、进度达成面板三种可复用骨架。

## 功能背景/动机

现有脚手架已有 `208-onboarding-welcome-layout`（欢迎页/引导页/快速设置向导），但它专注于"首次启动"的引导流程。在桌面应用中，用户引导不仅限于第一天，还包括：功能发现（逐步展示高级功能）、任务清单（帮助用户完成关键设置）、以及进度达成（激励用户探索更多功能）。这些场景共享相似的结构：可勾选的任务项、进度指示、完成奖励/反馈、以及跳过/稍后提醒的选项。

本功能提供三种引导检查清单的布局模板，与 `208` 形成互补，覆盖从首次使用到深度探索的完整用户引导生命周期。

## 功能描述

包含以下三种引导检查清单布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `NewUserTaskListLayout` | 新用户任务清单：侧边进度 + 任务列表 + 完成动画 | 首次使用引导、关键功能激活 |
| `FeatureDiscoveryLayout` | 功能发现引导：高亮提示 + 步骤标注 + 下一步按钮 | 新功能上线、高级功能发现 |
| `ProgressAchievementLayout` | 进度达成面板：环形进度 + 已解锁/待解锁功能网格 | 用户激励、功能探索引导 |

每种布局均内建：
- 任务完成状态（未开始/进行中/已完成）
- 进度指示（进度条或环形图）
- 完成反馈（勾选动画、庆祝效果占位）
- 跳过/稍后提醒选项
- 与 `Checkbox`、`Progress`、`Card`、`Empty` 的衔接

## 目标用户

- 需要提升新用户留存和功能发现的开发者
- 希望以游戏化方式引导用户探索应用的开发者

### 涉及的技术点

- 使用现有 `Progress`、`Checkbox`、`Card`、`Button` 组件
- 使用 `ui/circular-progress`（如有）或 SVG 实现环形进度
- 使用 `AnimatedTransition` 实现任务完成的勾选动画和进度变化动画
- 使用 `Badge` 展示"新"、"推荐"等标签
- 使用 `Empty` 组件处理所有任务完成后的状态（"恭喜！您已完成所有引导任务"）

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/NewUserTaskListLayout.vue`
- 新增 `src/components/layouts/FeatureDiscoveryLayout.vue`
- 新增 `src/components/layouts/ProgressAchievementLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- `NewUserTaskListLayout` 可与 `WelcomeHeroLayout`（208）组合（欢迎页底部的任务清单）
- `FeatureDiscoveryLayout` 可与 `OnboardingCarouselLayout`（208）组合（轮播式的功能发现）
- `ProgressAchievementLayout` 可与 `DashboardLayout` 组合（仪表盘内的探索进度 widget）

**Props 设计（以 NewUserTaskListLayout 为例）：**

```ts
interface OnboardingTask {
  id: string
  title: string
  description?: string
  completed: boolean
  actionLabel?: string      // "去完成"按钮文案
  skippable?: boolean       // 是否可跳过，默认 true
}

interface NewUserTaskListLayoutProps {
  tasks: OnboardingTask[]
  title?: string
  showProgress?: boolean    // 是否显示进度条，默认 true
  allowSkipAll?: boolean    // 是否允许"跳过全部"，默认 true
}

// Events
// @task-click   — 点击某个任务的去完成按钮
// @task-complete — 标记任务完成
// @skip-all     — 跳过全部引导
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `NewUserTaskListLayout` 的任务完成时，勾选图标有缩放弹跳动画（200ms）
- [ ] 进度条在任务状态变化时有平滑的宽度过渡动画
- [ ] `FeatureDiscoveryLayout` 支持步骤导航，最后一步的"下一步"变为"开始使用"
- [ ] `ProgressAchievementLayout` 的待解锁卡片使用 `opacity-50` + 锁图标，与已解锁形成对比
- [ ] 所有任务完成后，自动展示成功状态（`ResultSuccessLayout` 或自定义庆祝动画占位）
- [ ] 跳过引导的状态持久化到 `localStorage`，下次不再展示
- [ ] 暗色模式下进度环和卡片的高亮色使用主题 `primary` token

## 优先级

P1 — 用户引导对新用户留存和深度功能发现至关重要。
