# 223 仪表板数据面板布局（Dashboard Data Panel Layout）

## 功能名称

仪表板数据面板布局 — 提供复杂仪表盘、实时监控面板、个人工作台三种可复用骨架。

## 功能背景/动机

现有脚手架已有 `DashboardLayout.vue`（基础侧边栏 + 统计卡片 + 最近活动列表），但它是单一的基础示例，缺少**可复用的、模块化的仪表盘布局模板**。在实际桌面应用中，仪表盘的需求非常多样：不同尺寸的 widget 拼接、可拖拽调整的面板、实时数据刷新状态、以及个人工作台的快捷入口组合。

本功能在现有 `DashboardLayout` 基础上，提供更复杂的仪表盘布局变体，强调 widget 的模块化、面板的可组合性，以及与 `ChartContainer`、`MetricCards` 等组件的衔接。

## 功能描述

包含以下三种仪表盘布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `ComplexDashboardLayout` | 多区域仪表盘：顶部 KPI + 中部图表区 + 底部活动/列表区 | 管理后台首页、数据分析概览 |
| `RealtimeMonitorLayout` | 实时监控面板：大字号指标 + 实时图表 + 警报列表 + 刷新状态 | 系统监控、服务器状态、实时流量 |
| `PersonalWorkspaceLayout` | 个人工作台：快捷入口 + 待办概览 + 最近访问 + 收藏夹 | 个人效率工具、工作台首页 |

每种布局均内建：
- 区域化的模块结构（每个区域有标题和操作按钮占位）
- 刷新控制（自动刷新指示器、手动刷新按钮、最后更新时间）
- 空模块状态（某个 widget 无数据时的占位）
- 与 `KpiCardsLayout`（214）、`ChartContainer`、`Timeline`（203）的衔接

## 目标用户

- 需要实现管理后台、数据概览、个人工作台的开发者
- 希望仪表盘模块可组合、可扩展的开发者

### 涉及的技术点

- 使用 CSS Grid 的 `grid-template-columns` 和 `grid-template-rows` 实现模块化的区域划分
- 使用现有 `Card`、`ChartContainer`、`Badge`、`Avatar` 构建各模块内容
- 使用 `ui/skeleton/Skeleton.vue` 作为各模块的加载占位
- 使用 `Progress` 或 `Spinner` 展示实时刷新状态
- 使用 `ScrollArea` 处理活动列表和警报列表的独立滚动

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/ComplexDashboardLayout.vue`
- 新增 `src/components/layouts/RealtimeMonitorLayout.vue`
- 新增 `src/components/layouts/PersonalWorkspaceLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出
- 可与 `KpiCardsLayout`（214）、`MetricTrendLayout`（214）组合（替换内置的 KPI 区域）
- 可与 `TimelineLayout`（203）组合（仪表盘内的活动时间线）

**Props 设计（以 ComplexDashboardLayout 为例）：**

```ts
interface DashboardWidget {
  id: string
  title: string
  area: 'kpi' | 'chart-left' | 'chart-right' | 'activity' | 'quick-actions'
}

interface ComplexDashboardLayoutProps {
  widgets?: DashboardWidget[]
  lastUpdated?: Date
  autoRefresh?: boolean
  refreshInterval?: number   // 秒
  loading?: boolean
}

// Slot 约定
// #kpi           — KPI 区域内容
// #chart-left    — 左侧图表区
// #chart-right   — 右侧图表区
// #activity      — 活动列表区
// #quick-actions — 快捷操作区
// #header-extra  — 头部额外操作
```

## 验收标准

- [ ] 三种布局均能在 `ComponentPlayground` 中独立预览
- [ ] `ComplexDashboardLayout` 的各模块在窗口宽度变化时自动重排：大屏 2x2 → 中屏 1x2 → 小屏 1x1
- [ ] `RealtimeMonitorLayout` 的实时指标支持通过 `autoRefresh` 控制自动刷新，刷新时有视觉脉冲指示
- [ ] `PersonalWorkspaceLayout` 的快捷入口按钮支持通过 slot 自定义图标和文案
- [ ] 每个模块标题栏支持通过 `#widget-header-{id}` slot 自定义操作按钮
- [ ] 空模块自动展示 `Empty` 占位，不破坏整体网格布局
- [ ] 暗色模式下图表和卡片的边框对比度适配
- [ ] 所有布局的刷新状态和时间戳支持通过 props 传入，不绑定具体数据源

## 优先级

P1 — 仪表盘是桌面应用的"门面"，直接影响用户的第一印象。
