# 200 表单布局模板（Form Layout Templates）

## 功能名称

表单布局模板 — 提供单列、双列、分组（FieldSet）和步骤式 Wizard 四种可复用表单页面骨架。

## 功能背景/动机

在桌面应用中，表单是数据录入、配置管理和内容编辑的核心界面。现有脚手架已具备丰富的原子表单组件（`Input`、`Select`、`Switch`、`Field`、`Form` 等），但缺少**页面级的表单布局编排模板**。开发者每次新建表单页面都需要重新设计字段排列、分组逻辑和按钮区位置，重复劳动多且风格难以统一。

本功能提供一套即插即用的表单页面级 `Layout` 组件，让开发者只需关注业务字段，无需关心布局排版。

## 功能描述

包含以下四种表单布局变体：

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| `FormSingleColumnLayout` | 单列垂直排列，标签在上 | 简单录入、登录、搜索筛选 |
| `FormTwoColumnLayout` | 双列网格排列，标签在上或左对齐 | 复杂配置、人员信息、设备参数 |
| `FormGroupedLayout` | 多卡片/FieldSet 分组，每组可折叠 | 大量字段分类、系统设置向导 |
| `FormWizardLayout` | 顶部 Stepper 步骤指示器 + 分步内容区 | 注册流程、安装向导、分步审批 |

每种布局均内建：
- 统一的页面标题区（标题 + 副标题 + 可选返回按钮）
- 统一的表单操作栏（保存/取消/重置，吸底或表单末尾）
- 与 `Field`、`FormItem` 的无缝衔接
- 响应式降级策略（双列 → 单列）

## 目标用户

- 使用本脚手架开发桌面应用的开发者
- 需要快速搭建配置页、录入页、向导页的场景

## 详细设计

### 布局结构描述

#### 1. 单列表单（FormSingleColumnLayout）

```
+--------------------------------------------------+
| ← 返回    标题                                    |
|           副标题描述                              |
+--------------------------------------------------+
|                                                  |
|  ┌──────────────────────────────────────────┐   |
|  │ Label                                    │   |
|  │ [          Input                         ]│   |
|  │ Hint text                                │   |
|  └──────────────────────────────────────────┘   |
|  ┌──────────────────────────────────────────┐   |
|  │ Label                                    │   |
|  │ [          Input                         ]│   |
|  └──────────────────────────────────────────┘   |
|                                                  |
|  ┌──────────────────────────────────────────┐   |
|  │  [  取消  ]  [      保存      ]           │   |
|  └──────────────────────────────────────────┘   |
+--------------------------------------------------+
```

#### 2. 双列表单（FormTwoColumnLayout）

```
+--------------------------------------------------+
| ← 返回    标题                                    |
+--------------------------------------------------+
|  ┌──────────────┐  ┌──────────────┐             |
|  │ Label        │  │ Label        │             |
|  │ [ Input    ] │  │ [ Input    ] │             |
|  └──────────────┘  └──────────────┘             |
|  ┌──────────────┐  ┌──────────────┐             |
|  │ Label        │  │ Label        │             |
|  │ [ Input    ] │  │ [ Input    ] │             |
|  └──────────────┘  └──────────────┘             |
|  ┌──────────────────────────────────────────┐   |
|  │ 全宽字段（如 Textarea）                  │   |
|  └──────────────────────────────────────────┘   |
|                               [取消] [保存]      |
+--------------------------------------------------+
```

#### 3. 分组表单（FormGroupedLayout）

```
+--------------------------------------------------+
| ← 返回    标题                                    |
+--------------------------------------------------+
| ▼ 基本信息                                        |
|  ┌──────────────────────────────────────────┐   |
|  │  双列字段...                             │   |
|  └──────────────────────────────────────────┘   |
| ▶ 高级设置（可折叠）                              |
| ▶ 通知偏好（可折叠）                              |
|                                                  |
|                               [取消] [保存]      |
+--------------------------------------------------+
```

#### 4. 步骤式 Wizard（FormWizardLayout）

```
+--------------------------------------------------+
|  [ 1 ]────[ 2 ]────[ 3 ]────[ 4 ]                |
|  基本信息   配置项    确认      完成               |
+--------------------------------------------------+
|                                                  |
|  （当前步骤内容区，可嵌入上述任意子布局）          |
|                                                  |
+--------------------------------------------------+
|  [  上一步  ]              [  下一步  ]           |
+--------------------------------------------------+
```

### 涉及的技术点

- 使用现有 `Field`、`FieldSet`、`FormItem` 组件完成字段包裹和错误展示
- 使用现有 `Stepper` 组件完成 Wizard 步骤指示器
- 使用现有 `Card`、`Accordion` 组件完成分组表单的可折叠区块
- 使用 `ResizablePanelGroup` 实现双列表单中左右区域的拖拽调整（可选增强）
- 使用 Tailwind CSS `grid-cols-1 md:grid-cols-2` 实现响应式双列

### 与现有 layouts/ 的衔接

- 新增 `src/components/layouts/FormSingleColumnLayout.vue`
- 新增 `src/components/layouts/FormTwoColumnLayout.vue`
- 新增 `src/components/layouts/FormGroupedLayout.vue`
- 新增 `src/components/layouts/FormWizardLayout.vue`
- 在 `src/components/layouts/index.ts` 中导出以上组件
- 各布局接受 `<slot>` 作为表单内容，不绑定具体表单控件

### 需要新增/修改的文件

```
desktop-app/src/components/layouts/
  ├── FormSingleColumnLayout.vue    # 单列表单骨架
  ├── FormTwoColumnLayout.vue       # 双列表单骨架
  ├── FormGroupedLayout.vue         # 分组表单骨架
  ├── FormWizardLayout.vue          # 步骤式向导骨架
  └── index.ts                      # 追加导出
```

**Props 设计（以 FormWizardLayout 为例）：**

```ts
interface WizardStep {
  id: string
  title: string
  description?: string
  validate?: () => boolean | Promise<boolean>
}

interface FormWizardLayoutProps {
  steps: WizardStep[]
  currentStep: number
  showSteps?: boolean
  confirmText?: string
  cancelText?: string
}
```

## 验收标准

- [ ] 四种布局均能在 `ComponentPlayground` 中独立预览
- [ ] 单列表单在窗口宽度 < 640px 时自动全宽，无横向滚动
- [ ] 双列表单在窗口宽度 < 768px 时自动降级为单列
- [ ] 分组表单的每个分组均可通过 props 控制默认展开/收起
- [ ] Wizard 布局支持步骤验证拦截，验证失败时阻止进入下一步
- [ ] 所有布局的操作按钮区支持自定义 slot（`#actions`）
- [ ] 与现有 `Field`、`FormItem` 的错误提示样式无缝兼容

## 优先级

P0 — 表单是桌面应用最高频的页面类型之一，对脚手架核心价值提升显著。
